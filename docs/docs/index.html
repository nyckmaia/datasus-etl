Documentation

Technical reference.

Everything a developer or data engineer needs to automate, embed, or extend DataSUS ETL. The desktop app is a convenience — the engine is the CLI, the Python API, and the partitioned parquet store underneath.

§ 01

CLI reference

datasus pipeline, update, status, db, ui. Shared flags, exit codes, scripting recipes.

§ 02

Python API

The same pipeline, imported as a library. Useful when you want the ETL inside a larger workflow (Airflow, Prefect, notebook-driven analysis).

§ 03

Architecture

How the pieces fit. Path resolution, legacy migration, SPA bundling, tkinter-in-subprocess, and the release pipeline.

§ 04

Subsystem schemas

SIHSUS and SIM column lists, types, enrichments. What is identical to the DATASUS original, what has been cleaned, and where you can read the SQL that did it.
