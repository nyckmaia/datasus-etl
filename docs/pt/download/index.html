Instaladores sem assinatura

Um clique extra, na primeira vez.

Assinatura digital custa dinheiro; o projeto ainda não comporta esse custo. Na primeira execução:

Windows
O SmartScreen mostra um banner azul. Clique em Mais informações, depois Executar assim mesmo.
macOS
O Gatekeeper bloqueia a primeira abertura. Clique com o botão direito sobre o app em /Aplicativos e escolha Abrir; confirme. Nas próximas vezes, abre normalmente.
Linux
Dê permissão de execução ao AppImage: chmod +x, depois clique duas vezes ou rode direto.
Usuários de Mac Intel

Sem instalador nativo —
compile do código.

O arquivo .dmg nativo é entregue apenas para Macs com Apple Silicon (chips da série M). Se o seu Mac for Intel, siga os passos ao lado para instalar a partir do código-fonte — funciona igual, usa exatamente o mesmo software, e leva uns cinco minutos de Terminal. Sem exigir conhecimento de programação.

Como saber

Clique na maçã da Apple → Sobre Este Mac. Se aparecer "Apple M1/M2/M3/M4…", use o DMG da tabela acima. Se aparecer "Intel Core i…", siga os passos ao lado.

Passo a passo

  1. Instale o Python 3.11 ou mais novo. Abra a página de downloads do python.org, baixe o instalador mais recente 3.11.x ou 3.12.x (o macOS 64-bit universal2) e execute. Clique em Continuar, Aceitar, Instalar.
  2. Abra o Terminal. Pressione ⌘+Espaço, digite Terminal, pressione Return. Uma janela preta e branca vai aparecer. É nela que você vai digitar os próximos comandos.
  3. Copie o projeto para o seu Mac. Cole esta linha no Terminal e pressione Return: 
    git clone https://github.com/nyckmaia/datasus-etl.git
    Se o macOS perguntar se quer instalar as "Developer Command Line Tools", clique em Install e espere terminar; depois rode o comando acima novamente.
  4. Entre na pasta do projeto. 
    cd datasus-etl
  5. Crie um ambiente Python isolado (mantém as dependências do app separadas do resto do sistema): 
    python3 -m venv .venv
source .venv/bin/activate
    O prompt do Terminal deve passar a começar com (.venv).
  6. Instale o aplicativo. Aqui o comando baixa e compila todas as dependências — leva 2–5 minutos dependendo da sua conexão. 
    pip install .
  7. Abra o aplicativo. 
    datasus ui
    Seu navegador padrão vai abrir em http://127.0.0.1:8787 — pronto. Para encerrar o servidor, volte ao Terminal e pressione Ctrl+C.
  8. Nas próximas vezes que quiser usar, abra o Terminal e rode: 
    cd datasus-etl
source .venv/bin/activate
datasus ui
Travou em algum passo?

Se um passo falhar, copie a mensagem de erro exata e abra uma issue no GitHub. Inclua a linha do Terminal que falhou — ajudamos a resolver.

Técnico

O que vem no pacote

Cada instalador empacota o mesmo binário compilado com Nuitka — o programa é idêntico entre plataformas. O instalador copia os arquivos para o diretório padrão do SO, cria um atalho que executa datasus ui e, se o checkbox estiver marcado, adiciona o comando datasus ao seu PATH para que comandos como datasus pipeline funcionem em qualquer terminal.

Na instalação

  • Windows → %ProgramFiles%\DataSUS ETL\ + atalho no Menu Iniciar + atualização opcional do PATH (HKCU).
  • macOS → /Applications/DataSUS ETL.app + link simbólico opcional em /usr/local/bin/datasus.
  • Linux → o AppImage é autossuficiente. Rode de qualquer lugar. Uma entrada .desktop é registrada automaticamente quando você coloca o arquivo em ~/.local/share/applications/.

Na primeira abertura

O app sobe um servidor web local em 127.0.0.1:8787 e abre seu navegador padrão. O servidor é loopback — nada é exposto para a rede.

Configurações do usuário

A pasta de dados e outras configurações ficam salvas em ~/.config/datasus-etl/config.toml (Linux/macOS) ou em %APPDATA% no Windows.
