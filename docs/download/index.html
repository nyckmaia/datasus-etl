Unsigned installers

A one-time click.

Code-signing certificates cost money; this project doesn't carry that cost yet. The first time you run the installer:

Windows
SmartScreen shows a blue banner. Click More info, then Run anyway.
macOS
Gatekeeper blocks the first launch. Right-click the app in /Applications and choose Open; confirm. Subsequent launches work normally.
Linux
AppImages need the executable bit: chmod +x the file, then double-click or run it directly.
Intel Mac users

No native installer —
build from source.

We ship a native .dmg only for Macs with Apple Silicon (M-series chips). If your Mac has an Intel processor, follow the steps on the right to install from source — it works, it uses exactly the same code, and takes about five minutes of terminal time. No programming experience required.

How to check

Click the Apple logo → About This Mac. If it says "Apple M1/M2/M3/M4…", use the DMG in the table above. If it says "Intel Core i…", follow the steps on the right.

Step-by-step

  1. Install Python 3.11 or newer. Open the python.org downloads page, grab the latest 3.11.x or 3.12.x installer (the macOS 64-bit universal2 one), and run it. Click Continue, Agree, Install.
  2. Open the Terminal app. Press ⌘+Space, type Terminal, press Return. A black-and-white window will appear. That's where you'll type the next commands.
  3. Copy the project to your Mac. Paste this line into Terminal and press Return: 
    git clone https://github.com/nyckmaia/datasus-etl.git
    If macOS asks whether to install "Developer Command Line Tools", click Install and wait for it to finish, then re-run the command above.
  4. Enter the project folder. 
    cd datasus-etl
  5. Create an isolated Python environment (keeps the app's dependencies away from the rest of your system): 
    python3 -m venv .venv
source .venv/bin/activate
    Your Terminal prompt should now start with (.venv).
  6. Install the app. This downloads and builds all the dependencies — takes 2–5 minutes depending on your connection. 
    pip install .
  7. Launch the app. 
    datasus ui
    Your default browser will open at http://127.0.0.1:8787 — you're in. To stop the server, come back to the Terminal and press Ctrl+C.
  8. Next time you want to use the app, open Terminal and run: 
    cd datasus-etl
source .venv/bin/activate
datasus ui
Hit a snag?

If a step fails, copy the exact error message and open an issue on GitHub. Include the line of your Terminal that failed — we'll help.

Technical

Package contents

Each installer bundles the same Nuitka-compiled binary — the app you run is the same one across platforms. The installer places the files in the OS-standard location, creates a shortcut to launch datasus ui, and (if you leave the checkbox marked) adds the datasus CLI to your PATH so commands like datasus pipeline work from any terminal.

On install

  • Windows → %ProgramFiles%\DataSUS ETL\ + Start Menu shortcut + optional PATH update (HKCU).
  • macOS → /Applications/DataSUS ETL.app + optional symlink in /usr/local/bin/datasus.
  • Linux → AppImage is self-contained. Run it from anywhere. A .desktop entry registers automatically when you place it in ~/.local/share/applications/.

On first launch

The app starts a local web server on 127.0.0.1:8787 and opens your default browser. The server is loopback-only — nothing is exposed to the network.

User settings

Your data-directory preference and other settings are persisted in ~/.config/datasus-etl/config.toml (Linux/macOS) or the equivalent location under %APPDATA% on Windows.
